          User Acceptance Plan


          LANDING PAGE

          The user starts by entering the index.html which is the Landing Page. By pressing the Twitter icon or Facebook icon the user will enter our Official Twitter Site or Facebook Page where we grant more information about the web app. If the user presses the Github icon he/she will be transported to our Repository page and by pressing the "START THE QUIZ" button the user will be taken the app.html which is the Official Web App Page. The user can also interact with the Twitter embed under the "START THE QUIZ" button.


          WEB APP


          CHOOSE A TREASURE HUNT

          When the user proceeds to the WEB APP there is a button "Select a Treasure Hunt" which displays treasure hunts to select. Additionally, there is a "Home" button that if it's clicked it takes the user in the Landing Page. Also, there is a Barcode icon and if the user presses it the device's camera will be opened and can scan a QR Code. By pressing the switch camera button it switches the camera and by pressing the "Go Back" button it takes the user on the previous page. If the user clicks an available Treasure Hunt he will be taken to the next page.


          CHOOSE A USERNAME

          After that he will see a text box which if he clicks he can enter his username and then press the "START" button to proceed to the next page. If he doesn't enter a username an error will be shown saying that the username is missing. If he enters a username which is already taken an error will be shown saying that username is already taken.


          QUESTIONS PAGE

          Then the user will be taken in the Questions Page in which he will start taking the quiz. There are different types of questions. If the user answers correctly he will be taken to the next question but if he answers wrong he will remain in the current question question until he answers correctly or skips the question. If a question can be skipped a "SKIP" button will be available which takes the user to the next question if it is pressed.

          There is also a Location icon which refreshes the user's current Location. The user can interact with the map to change the view of his location

          1 type of question. The user must press the text box, type some text and then press submit to Submit his answer.

          2 type of question. The user must press the text box, type number(s) and then press submit to Submit his answer.

          3 type of question. The user must press the option A or B or C or D and then press submit to Submit his answer.

          4 type of question. The user must press True or False and then press the Submit to submit his answer.

          After the user completes the last question he will be taken to the Leaderboard Page


          LEADERBOARD PAGE

          The Leaderbord will be shown and user can press the Follow Twitter button to follow our Twitter Page.



          Nielsen's 10 Heuristics


          Heuristic What We Did
          1. Visibility of system status Added a loading animation when the user chooses his username in the treasure hunt.
          2. Match between system and the real world The app is user friendly since it has simple structure and is giving the user simple instructions to read.
          3. User control and freedom If the user accidentally exits the app he will be able to resume his progress when he re enters the gamwe.
          4. Consistency and standards When the user starts the app, the app takes the user to the next step until he reaches the end and the styling of the pages is consistent.
          5. Error Prevention If the user accidentally press the SKIP button an alert will show asking the user to confirm his choice.
          6. Recognition rather than recall The instructions in the beginning help the user understand the game and the user's details and score will appear through the game.
          7. Flexibility and efficiency of use 1 question requires location so when the user gets to that question the app automatically asks the user to have access to his location.
          8. Aesthetic and minimalist design The Barcode Scanner can be accessed at any point by just click it's icon.
          9. Help users recognize, diagnose and recover from errors If an error occurs the user will be notified with an alert explaining briefly the error.
          10. Help and documentation The steps are clear and the procedure is simple enough to not cause frustration to the user.
